## 1. 前提・スコープ

### 既存バトルシステム（現状）

```
BattleEngine (game/battle.py)
├── player_action()     : プレイヤー操作（攻撃/スキル/防御/アイテム）
├── enemy_action()      : 敵ターン（全生存敵がヘイト比率でターゲット選択 → 通常攻撃のみ）
├── buffs dict          : バフ/デバフ/状態異常を session_state と共有
├── hate dict           : ヘイト値（ターゲット選択の重み）
└── tick_buffs()        : ターン終了時のカウントダウン・毒ダメージ
```

### 現在の `enemy_action()` の課題

| 項目 | 現状 | 課題 |
|---|---|---|
| 行動パターン | 通常攻撃のみ | 単調で戦略性がない |
| ターゲット選択 | ヘイト比率の確率選択 | HPが低いキャラを狙わない |
| HP危機対応 | なし | 敵が追い詰められても何も変わらない |
| ボス専用行動 | なし | ボスも雑魚も同じ行動 |

## 2. R-12 敵AI（疑似AI）

### 2.1 設計方針

- **実装方式**: `config.py` に `ENEMY_AI_ACTIONS` 辞書を定義し、敵名をキーとするルールベース  
- **DBへの追加は行わない**（敵スキルテーブルは新設しない）  
- **BattleEngine 内に `EnemyAI` クラスを追加**し、`enemy_action()` から呼び出す形に置き換える  
- 既存のヘイト/挑発/状態異常チェックは維持する

### 2.2 敵AIの行動フロー

```
enemy_action() の内部（敵ごとにループ）
│
├─ ① スタンチェック → スキップ
│
├─ ② フェーズ判定
│    max_hp の何 % か計算
│    > 50% → NORMAL フェーズ
│    ≤ 50% → DANGER フェーズ
│
├─ ③ WIN_FIRST 判定（DANGER 時のみ）
│    生存パーティに HP ≤ 15% のキャラがいる？
│    → ターゲット選択を「最低HPキャラ優先」に変更
│
├─ ④ 行動選択（EnemyAI.choose_action）
│    enemy 名 → ENEMY_AI_ACTIONS でルックアップ
│    見つからなければ "default" ルールを使用
│    フェーズ別ローテーション（インデックス管理）で行動を決定
│
└─ ⑤ 行動実行（attack / skill / buff / debuff / status）
     BattleEngine の既存メソッドを再利用
```

### 2.3 行動フェーズ定義

| フェーズ | 判定条件 | 行動方針 |
|---|---|---|
| `NORMAL` | HP > max_hp × 0.5 | ローテーション通りに行動（攻撃中心） |
| `DANGER` | HP ≤ max_hp × 0.5 | 回復・妨害・防御バフ優先。なければ攻撃 |
| `WIN_FIRST` | DANGER かつ 味方に HP ≤ 15% のキャラ存在 | ターゲットを最低HPキャラ固定 + DANGER 行動 |

### 2.5 敵最大HP追跡

`Enemy` クラスには `max_hp` がない（`hp` を初期値として扱っている）。  
実装時は **`BattleEngine` が敵最大HPを辞書で管理**する方式を採用する。

```python
class BattleEngine:
    def __init__(self, ...):
        ...
        # 敵の最大HPを初期化時に記録
        self.enemy_max_hp: dict[int, int] = {e.id: e.hp for e in enemies}
        # 敵ごとのローテーションインデックス
        self.enemy_rotation_idx: dict[int, int] = {e.id: 0 for e in enemies}
```

- `enemy_max_hp` は session_state に `"battle_enemy_max_hp"` キーで保存する（BattleEngine の他フィールドと同様）
- `enemy_rotation_idx` も同様に `"battle_enemy_rotation"` キーで保存する

### 2.6 ターゲット選択ルール（拡張）

`_select_target()` は変更なし。`WIN_FIRST` 判定は呼び出し側（`_execute_enemy_action()`）で行う。

```
WIN_FIRST ターゲット選択:
  生存パーティの中で hp / max_hp が最小のキャラを返す
  （同率なら先頭キャラ）
```

### 2.7 実装詳細

#### 新規メソッド: `EnemyAI` クラス（`game/battle.py` 内に追加）

```python
class EnemyAI:
    """敵AIのフェーズ判定・行動選択を担う内部クラス"""

    @staticmethod
    def get_phase(enemy: Enemy, max_hp: int) -> str:
        """NORMAL / DANGER を返す"""
        ratio = enemy.hp / max(1, max_hp)
        return "DANGER" if ratio <= 0.5 else "NORMAL"

    @staticmethod
    def is_win_first(party: list[Character]) -> bool:
        """瀕死のパーティメンバーがいるか（HP ≤ 15%）"""
        for c in party:
            if c.is_alive() and c.max_hp > 0 and c.hp / c.max_hp <= 0.15:
                return True
        return False

    @staticmethod
    def select_win_first_target(party: list[Character]) -> "Character | None":
        """最低HP率の生存キャラを返す"""
        alive = [c for c in party if c.is_alive()]
        if not alive:
            return None
        return min(alive, key=lambda c: c.hp / max(1, c.max_hp))

    @staticmethod
    def choose_action(enemy: Enemy, phase: str, rotation_idx: int) -> tuple[str, dict]:
        """
        行動名と効果定義を返す。
        Returns:
            (action_name: str, action_def: dict)
        """
        from config import ENEMY_AI_ACTIONS
        pattern = ENEMY_AI_ACTIONS.get(enemy.name, ENEMY_AI_ACTIONS["default"])

        if phase == "DANGER":
            priority = pattern.get("danger_priority", ["attack"])
            # danger_priority は優先順リスト: ローテーション制でなく先頭から試行
            # 制約: heal_self / buff は MP 不要なので常に実行可。
            #        status 系はランダムに選択して変化をつける
            action_name = priority[rotation_idx % len(priority)]
        else:
            rotation = pattern.get("normal_rotation", ["attack"])
            action_name = rotation[rotation_idx % len(rotation)]

        action_def = pattern.get("actions", {}).get(action_name, {"type": "attack"})
        return action_name, action_def
```

#### 変更: `enemy_action()` の置き換え

```python
def enemy_action(self) -> list[str]:
    """全生存敵がAIに基づいて行動する（R-12 ルールベースAI）"""
    messages: list[str] = []
    if not any(c.is_alive() for c in self.party):
        return messages

    for enemy in self.enemies:
        if not enemy.is_alive():
            continue
        if self.has_status(enemy, "stun"):
            messages.append(f"{enemy.name} はスタン状態で行動できない！")
            continue

        max_hp   = self.enemy_max_hp.get(enemy.id, enemy.hp)
        phase    = EnemyAI.get_phase(enemy, max_hp)
        rot_idx  = self.enemy_rotation_idx.get(enemy.id, 0)

        action_name, action_def = EnemyAI.choose_action(enemy, phase, rot_idx)

        # ローテーションインデックスを進める
        self.enemy_rotation_idx[enemy.id] = rot_idx + 1

        # WIN_FIRST チェック（DANGER 時のみ）
        if phase == "DANGER" and EnemyAI.is_win_first(self.party):
            forced_target = EnemyAI.select_win_first_target(self.party)
        else:
            forced_target = None

        msg = self._execute_enemy_action(
            enemy, action_name, action_def, forced_target
        )
        messages.append(msg)

    self._defending.clear()
    self.turn += 1
    return messages
```

#### 新規メソッド: `_execute_enemy_action()`

```python
def _execute_enemy_action(
    self,
    enemy: Enemy,
    action_name: str,
    action_def: dict,
    forced_target=None,
) -> str:
    atype      = action_def.get("type", "attack")
    power_rate = action_def.get("power_rate", 1.0)
    max_hp     = self.enemy_max_hp.get(enemy.id, enemy.hp)

    if atype == "attack":
        target = forced_target or self._select_target()
        if target is None:
            return f"{enemy.name} は様子を見ている…"
        atk   = self.get_effective_attack(enemy)
        def_  = self.get_effective_defense(target)
        if target.id in self._defending:
            def_ *= 2
        dmg    = self.calc_damage(int(atk * power_rate), def_)
        actual = target.take_damage(dmg)
        label  = "" if power_rate == 1.0 else ("強烈な" if power_rate > 1.2 else "弱い")
        msg    = f"{enemy.name} の{label}攻撃！ {target.name} に {actual} のダメージ！"
        if not target.is_alive():
            msg += f" {target.name} は倒れた…"
        return msg

    elif atype == "attack_all":
        alive_targets = [c for c in self.party if c.is_alive()]
        if not alive_targets:
            return f"{enemy.name} は様子を見ている…"
        atk  = self.get_effective_attack(enemy)
        msgs = []
        for t in alive_targets:
            def_  = self.get_effective_defense(t)
            if t.id in self._defending:
                def_ *= 2
            dmg    = self.calc_damage(int(atk * power_rate), def_)
            actual = t.take_damage(dmg)
            part   = f"{t.name} に {actual}"
            if not t.is_alive():
                part += "(戦闘不能)"
            msgs.append(part)
        return f"{enemy.name} の全体攻撃！ " + " / ".join(msgs)

    elif atype == "status":
        target = forced_target or self._select_target()
        if target is None:
            return f"{enemy.name} は様子を見ている…"
        kind     = action_def.get("kind", "poison")
        duration = action_def.get("duration", 2)
        return f"{enemy.name} の特殊行動！ " + self.apply_status(target, kind, duration, action_name)

    elif atype == "status_all":
        alive_targets = [c for c in self.party if c.is_alive()]
        kind     = action_def.get("kind", "poison")
        duration = action_def.get("duration", 2)
        msgs = [self.apply_status(t, kind, duration, action_name) for t in alive_targets]
        return f"{enemy.name} の特殊行動！ " + " / ".join(msgs)

    elif atype == "buff":
        stat     = action_def.get("stat", "defense")
        amount   = action_def.get("amount", 5)
        duration = action_def.get("duration", 3)
        return f"{enemy.name} は力を高めた！ " + self.apply_buff(enemy, stat, amount, duration, action_name)

    elif atype == "heal_self":
        heal_amount = max(1, int(max_hp * power_rate))
        old_hp      = enemy.hp
        enemy.hp    = min(max_hp, enemy.hp + heal_amount)
        actual      = enemy.hp - old_hp
        return f"{enemy.name} は自己回復した！ HP が {actual} 回復！"

    return f"{enemy.name} は様子を見ている…"
```

## 3. R-13 味方自動行動AI

### 3.1 設計方針

- **オプション機能**：「自動行動」トグルを ON にした場合のみ全キャラが自動行動する  
- **ポリシー設定**：キャラクターごとに行動方針（攻撃重視/回復優先/防御重視）を選べる  
- **手動優先**：「自動行動」OFF 時は既存の完全手動操作を維持する（後退互換）  
- **スキル使用**：キャラクターのクラスに応じて DB からスキルを取得し、適切なものを選択する

### 3.2 行動方針（ポリシー）定義

| ポリシーキー | 表示名 | 行動方針 |
|---|---|---|
| `attack` | ⚔️ 攻撃重視 | 最も強い攻撃スキルを使用。MPなければ通常攻撃 |
| `heal` | 💚 回復優先 | パーティで HP% 最低のキャラを回復。全員 HP ≥ 70% なら攻撃 |
| `defend` | 🛡️ 防御重視 | 防御バフ/挑発を使用。なければ自身を防御。敵を倒せる場合は攻撃優先 |

`config.py` に定数を追加:

```python
# config.py に追加
ALLY_POLICIES: dict[str, str] = {
    "attack": "⚔️ 攻撃重視",
    "heal":   "💚 回復優先",
    "defend": "🛡️ 防御重視",
}
```

### 3.3 自動行動アルゴリズム

#### `BattleEngine.ally_auto_action()` 新規メソッド

```
ally_auto_action(character, policy, skills, party, enemies) -> str

入力:
  character  : 行動キャラクター
  policy     : "attack" | "heal" | "defend"
  skills     : DB から取得したキャラクターのスキルリスト
  party      : パーティ全員（生存/戦闘不能含む）
  enemies    : 生存敵リスト

出力:
  戦闘ログ文字列（player_action() の戻り値と同形式）
```

#### ポリシー別ロジック詳細

**`attack`（攻撃重視）**

```
1. スタンチェック → スキップ
2. 攻撃スキル（effect_type == "attack"）をMP順で最大MPのものを選択
   かつ character.mp >= skill.mp_cost
3. 対象 = HP最低の生存敵
4. 使えるスキルがなければ通常攻撃（対象は同じ）
```

**`heal`（回復優先）**

```
1. スタンチェック → スキップ
2. 沈黙チェック → スキルが使えない場合、通常攻撃へフォールバック
3. パーティのHP%を確認
   瀕死（HP ≤ 30%）のキャラがいる?
     YES → revive/heal スキルを最優先で使用（対象は最低HP%キャラ）
   HP < 70% のキャラがいる?
     YES → heal スキルを使用（対象は最低HP%キャラ）
   cure 対象がいる?（状態異常保持キャラ）
     YES → cure スキルを使用
4. 全員 HP ≥ 70% かつ異常なし → 攻撃スキルまたは通常攻撃へフォールバック
```

**`defend`（防御重視）**

```
1. スタンチェック → スキップ
2. 挑発スキルを保持かつ未使用 → 挑発を使用（effect_type == "buff_def" and name == "挑発"）
3. 防御バフスキルを保持 → 使用（効果が既に付与中はスキップ）
4. 敵の中に「あと1撃で倒せる」ものがある → 攻撃スキルまたは通常攻撃で仕留める
   判定: enemy.hp ≤ get_effective_attack(character) + 最大攻撃スキルpower - get_effective_defense(enemy)
5. 上記すべて不該当 → 防御行動（"defend"）
```

#### 状態異常への対応

```
スタン中   → 全ポリシー共通でスキップ（メッセージ返却）
沈黙中     → スキル使用不可 → 通常攻撃へフォールバック
毒/防御低下 → 回復優先ポリシーなら cure スキルを使用
```

### 3.5 実装詳細

#### `game/battle.py` への追加

```python
def ally_auto_action(
    self,
    character: "Character",
    policy: str,
    skills: list,
) -> str:
    """
    指定ポリシーに従ってキャラクターを自動行動させる。
    Returns: 戦闘ログ文字列
    """
    if not character.is_alive():
        return f"{character.name} は戦闘不能のため行動できない。"

    if self.has_status(character, "stun"):
        return f"{character.name} はスタン状態で行動できない！"

    alive_enemies = [e for e in self.enemies if e.is_alive()]
    alive_party   = [c for c in self.party if c.is_alive()]

    silenced = self.has_status(character, "silence")

    # ────────────────────────────────────────────
    # ユーティリティ: HP最低の敵 / 味方を取得
    # ────────────────────────────────────────────
    def weakest_enemy():
        return min(alive_enemies, key=lambda e: e.hp) if alive_enemies else None

    def weakest_ally():
        """HP% 最低の生存味方（自身を含む）"""
        return min(alive_party, key=lambda c: c.hp / max(1, c.max_hp)) if alive_party else character

    def get_skill(etypes: list[str], top_power: bool = True) -> "Skill | None":
        """指定 effect_type のスキルを MP が足りるものから選ぶ"""
        cands = [s for s in skills if s.effect_type in etypes and character.mp >= s.mp_cost]
        if not cands:
            return None
        return max(cands, key=lambda s: s.power) if top_power else cands[0]

    # ────────────────────────────────────────────
    # attack ポリシー
    # ────────────────────────────────────────────
    if policy == "attack":
        target = weakest_enemy()
        if target is None:
            return f"{character.name} は攻撃対象がいない。"
        if not silenced:
            sk = get_skill(["attack"])
            if sk:
                return self.player_action(character, "skill", target=target, skill=sk)
        return self.player_action(character, "attack", target=target)

    # ────────────────────────────────────────────
    # heal ポリシー
    # ────────────────────────────────────────────
    elif policy == "heal":
        # 戦闘不能メンバーがいれば蘇生
        if not silenced:
            dead_members = [c for c in self.party if not c.is_alive()]
            if dead_members:
                sk = get_skill(["cure"])  # cure スキルは蘇生ではないが revive アイテム相当
                # ※ 蘇生スキルは現状なし。cure で代替（heal フォールバック）
            # 瀕死 HP ≤ 30%
            critical = [c for c in alive_party if c.hp / max(1, c.max_hp) <= 0.3]
            if critical:
                sk = get_skill(["heal"])
                if sk:
                    return self.player_action(character, "skill", target=min(critical, key=lambda c: c.hp), skill=sk)
            # 状態異常保持キャラ
            ill = [c for c in alive_party if self._entity_key(c) in self.buffs and
                   any(b.get("stat") == "status" for b in self.buffs[self._entity_key(c)])]
            if ill:
                sk = get_skill(["cure"])
                if sk:
                    return self.player_action(character, "skill", target=ill[0], skill=sk)
            # HP < 70%
            hurt = [c for c in alive_party if c.hp / max(1, c.max_hp) < 0.7]
            if hurt:
                sk = get_skill(["heal"])
                if sk:
                    return self.player_action(character, "skill", target=min(hurt, key=lambda c: c.hp / max(1, c.max_hp)), skill=sk)
        # フォールバック: 攻撃
        target = weakest_enemy()
        if target is None:
            return f"{character.name} は行動対象がいない。"
        if not silenced:
            sk = get_skill(["attack"])
            if sk:
                return self.player_action(character, "skill", target=target, skill=sk)
        return self.player_action(character, "attack", target=target)

    # ────────────────────────────────────────────
    # defend ポリシー
    # ────────────────────────────────────────────
    elif policy == "defend":
        if not silenced:
            # 挑発優先
            taunt_sk = next((s for s in skills if s.name == "挑発" and character.mp >= s.mp_cost), None)
            if taunt_sk and not any(b.get("taunt") for b in self.buffs.get(self._entity_key(character), [])):
                return self.player_action(character, "skill", target=character, skill=taunt_sk)
            # 防御バフ（未適用のもの）
            def_sk = get_skill(["buff_def"])
            if def_sk:
                already = any(b.get("source") == def_sk.name for b in self.buffs.get(self._entity_key(character), []))
                if not already:
                    return self.player_action(character, "skill", target=character, skill=def_sk)
            # 止めを刺せる敵がいれば攻撃
            atk_val = self.get_effective_attack(character)
            one_shot = [
                e for e in alive_enemies
                if e.hp <= max(1, atk_val - self.get_effective_defense(e))
            ]
            if one_shot:
                sk = get_skill(["attack"])
                target = min(one_shot, key=lambda e: e.hp)
                if sk:
                    return self.player_action(character, "skill", target=target, skill=sk)
                return self.player_action(character, "attack", target=target)
        # 防御行動
        return self.player_action(character, "defend")

    # 不明なポリシー → 通常攻撃フォールバック
    target = weakest_enemy()
    if target is None:
        return f"{character.name} は行動対象がいない。"
    return self.player_action(character, "attack", target=target)
```

## 8. 後続タスク：味方AI知性値の細分化・成長システム拡張

### 閾値計算の変更（線形補間方式）

第7章の離散テーブル `ALLY_HEAL_THRESHOLDS` を **線形補間関数** に置き換える。

```python
# config.py から ALLY_HEAL_THRESHOLDS を削除し、以下の関数を game/battle.py に追加

def calc_heal_threshold(intelligence: int, key: str) -> float:
    """
    知性値（1〜10。10超は10扱い）から回復閾値を線形補間して返す。
    key: "critical" | "hurt"
    """
    BASES  = {"critical": 0.15, "hurt": 0.45}   # intelligence=1 時の最小値
    MAXES  = {"critical": 0.50, "hurt": 0.85}   # intelligence=10 時の最大値
    ratio  = (intelligence - 1) / 9             # 0.0 〜 1.0
    return BASES[key] + (MAXES[key] - BASES[key]) * ratio
```

### defend ポリシー：止め刺し判断の連続化

第7章では「知性値1: 判断なし / 知性値2: 1撃圏 / 知性値3: 2撃圏」という離散設計だったが、
10段階スケールでは **攻撃に切り替える "追撃係数"** を連続値で定義する。

```python
def calc_finish_multiplier(intelligence: int) -> float:
    """
    知性値（1〜10。10超は10扱い）に応じた「止め刺し判断の攻撃倍率上限」を返す。
    enemy.hp <= (atk - def) * multiplier のとき攻撃に転換。
    """
    # intel=1 → 0.8（1撃以下でも判断しない）
    # intel=5 → 1.0（ちょうど1撃で倒せる場合のみ）
    # intel=10 → 2.0（2撃圏内なら攻撃に転換）
    if intelligence <= 1:
        return 0.8
    return 0.8 + (intelligence - 1) * (1.2 / 9)
```

### 回復系スキルの INT ベース計算（案B）

回復系スキル（`effect_type="heal"`）の実回復量は、攻撃力ベースではなく
**INT ベース**で計算する。係数は `config.py` に定数として置く。

```python
HEAL_SKILL_INT_BASE_OFFSET = -4

heal_amount = (
    skill.power
    + HEAL_SKILL_INT_BASE_OFFSET
    + character.intelligence * HEAL_SKILL_INT_SCALE
) * heal_mult
```

- `character.intelligence` が `10` を超えている場合も、回復量計算ではそのまま反映する
- ポーションなどのアイテム回復は従来どおり固定値

### `level_up()` の変更

```python
# models/character.py（R-07 設計書の変更に追記）
def level_up(self, plan_key: str = "power") -> dict[str, int]:
    from config import LEVEL_UP_PLANS
    import random
    plan   = LEVEL_UP_PLANS.get(plan_key, LEVEL_UP_PLANS["power"])
    growth = plan["growth"]
    result = {}
    for stat, (lo, hi) in growth.items():
        delta = random.randint(lo, hi)
        if delta == 0:
            continue
        if stat == "intelligence":
            setattr(self, stat, getattr(self, stat, 0) + delta)
            result[stat] = delta
        else:
            setattr(self, stat, getattr(self, stat, 0) + delta)
            result[stat] = delta
    self.level += 1
    return result
```
