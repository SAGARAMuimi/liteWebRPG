## 1. 前提・スコープ

### 現在のダンジョン探索システム

```
DungeonManager (game/dungeon.py)
├── resolve_event()   : 部屋に入ったときのイベント解決
├── check_encounter() : エンカウント判定
├── get_random_enemies(): 敵生成
└── get_boss()        : ボス生成

pages/2_dungeon.py
├── current_room      : 部屋番号（0〜ROOMS_PER_FLOOR）を session_state で管理
├── [次の部屋へ進む]  : ボタンで current_room をインクリメント
└── ボス部屋           : room >= ROOMS_PER_FLOOR で判定

DungeonProgress (models/dungeon.py)
└── current_floor    : 現在の階層（整数）のみ
```

### 現在の線形進行の問題点

| 項目 | 現状 | 課題 |
|---|---|---|
| 移動方式 | 線形（部屋番号順） | 探索感がなく、選択の余地がない |
| マップ表示 | なし | 現在地が視覚的に把握できない |
| 方向選択 | なし | 「どこへ行くか」という意思決定がない |
| 行き止まり/迂回 | なし | 経路選択による戦略性がない |

### スコープ

- 既存の線形ダンジョン（`dungeon_id=1` 「旅立ちの洞窟」）は**そのまま残す**
- グリッドマップ方式の新ダンジョンを**追加**し、旅立ちの洞窟クリア後に解放する
- `dungeons` テーブルに `map_type`（`'linear'` / `'grid'`）カラムを追加してUIを分岐する
- 前後左右のテキストボタンで移動し、壁方向は `disabled=True`
- 現在地の周辺状況をテキストで描写する（テキストRPGのスタイルを維持）
- マップデータは `config.py` に静的定義する（DB非保存）
- 新規ファイル `game/map_manager.py` にマップロジックを分離する

### 後退互換への影響

既存の `DungeonProgress` は最初から `dungeon_id` FK で設計されており、  
複数ダンジョンを個別に管理できる構造が**すでに実装済み**である。

```python
# 現在の実装（変更不要）
DungeonProgress.get_or_create(db, user_id, dungeon_id)  # dungeon_idで分離済み
dungeon_id = st.session_state.get("current_dungeon_id", 1)  # 選択式の基盤あり
```

そのため、本要件は既存の線形ダンジョンを**一切変更せずに**グリッドダンジョンを追加できる。  
実装前に以下が安定していることを確認すること。

- R-02 バフ/デバフシステム
- R-04 状態異常システム
- R-05 アイテム（消耗品）
- R-08 分岐・イベントマス
- R-10 通貨・町機能
- R-11 装備システム
- R-12 / R-13 AI

## 2. 設計方針

### 基本方針

| 設計項目 | 採用方針 |
|---|---|
| ダンジョン追加方式 | 既存を残し、`dungeons` テーブルに `map_type` カラムで種別管理 |
| マップ表現 | 二次元グリッド（整数配列）。`config.py` に静的定義 |
| 座標系 | `(x, y)`、左上原点。東(+x) / 南(+y) が正方向 |
| 方向表現 | 絶対方向：北 / 南 / 東 / 西（テキストRPGスタイル） |
| ボス部屋 | ゴールマス（`CELL_GOAL`）に到達時に自動起動 |
| イベント | 既存 `EventResult` / `resolve_event()` を再利用 |
| UI分岐 | `map_type == 'linear'` → 既存UI、`'grid'` → グリッドUI |
| マップ永続化 | `dungeon_progress` の `current_x/y` で現在地のみDB保存 |
| ロック解除 | 既存の `DungeonProgress.is_cleared` フラグで判定（追加実装不要） |

### 既存システムとの関係

```
変更なし（線形ダンジョン）:
  2_dungeon.py → map_type == 'linear' → 既存UIをそのまま使用
  current_dungeon_id = 1（旅立ちの洞窟）

追加（グリッドダンジョン）:
  2_dungeon.py → map_type == 'grid'  → グリッドUIを使用
              → MapManager.move(direction)
              → DungeonManager.resolve_event_at(party, x, y)
  x, y = session_state["map_x"], session_state["map_y"]

ダンジョン選択:
  2_dungeon.py 冒頭で current_dungeon_id が未選択の場合に選択UIを表示
  旅立ちの洞窟クリア済み → グリッドダンジョンを解放表示
```

## 3. ダンジョン種別設計

### 3.1 dungeons テーブルへの map_type 追加

`dungeons` テーブルに `map_type` カラムを追加し、進行UIを切り替える基準とする。

```sql
-- data/db_init.sql に追加（マイグレーション SQL）
ALTER TABLE dungeons ADD COLUMN map_type VARCHAR(16) NOT NULL DEFAULT 'linear';
```

| map_type | 説明 | 対応ダンジョン |
|---|---|---|
| `'linear'` | 既存の部屋番号ベース進行 | 旅立ちの洞窟（id=1） |
| `'grid'` | X/Y座標グリッド移動 | 新規追加ダンジョン |

### 3.3 Dungeon モデルの変更

`models/dungeon.py` の `Dungeon` クラスに `map_type` カラムを追加する。

```python
# models/dungeon.py（変更箇所のみ）
class Dungeon(Base):
    __tablename__ = "dungeons"

    id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)
    name: Mapped[str] = mapped_column(String(64), nullable=False)
    floor: Mapped[int] = mapped_column(nullable=False)
    map_type: Mapped[str] = mapped_column(String(16), default="linear", nullable=False)  # ← 追加

    @property
    def is_grid(self) -> bool:  # ← 追加
        """グリッドマップ型ダンジョンかどうか"""
        return self.map_type == "grid"
```

## 4. マップデータ設計

### 4.1 セル種別定義

`config.py` に追加する定数。

```python
# R-14 セル種別
CELL_WALL    = 0  # 壁（移動不可）
CELL_PASSAGE = 1  # 通路（移動可能）
CELL_START   = 2  # スタート地点（各フロアの入場地点）
CELL_GOAL    = 3  # ボス部屋 / 次フロアへの扉（移動可能）
```

### 4.2 フロアマップ定義（config.py）

`FLOOR_MAPS: dict[int, dict]` を `config.py` に追加する。  
キーはグリッドダンジョン（`dungeon_id=2`）のフロア番号とする。

各フロアは以下のキーで構成される。

| キー | 型 | 説明 |
|---|---|---|
| `grid` | `list[list[int]]` | 二次元配列（行=Y軸、列=X軸） |
| `start` | `tuple[int, int]` | スタート座標 `(x, y)` |
| `goal` | `tuple[int, int]` | ゴール座標（ボス部屋） `(x, y)` |
| `fixed_events` | `dict[tuple, str]` | 座標 → イベント種別（省略時はランダム） |

```python
# config.py に追加

    "south": "南",
```

### 4.3 座標系・方向定義

```
   西(x-1) ←  → 東(x+1)
              ↑
           北(y-1)
              ↓
           南(y+1)
```

- グリッドは `grid[y][x]` の順でアクセスする。
- `y=0` が最北端、`y=max` が最南端。
- プレイヤーの「前後左右」ではなく絶対方向（北/南/東/西）で統一する。

## 5. MapManager クラス設計

新規ファイル `game/map_manager.py` に実装する。

### 5.1 クラス概要

```python
# game/map_manager.py
from __future__ import annotations
from config import (
    FLOOR_MAPS, DIRECTIONS, DIRECTION_NAMES_JA,
    CELL_WALL, CELL_DESCRIBE,
)


class MapManager:
    """
    グリッドマップ上のプレイヤー位置管理と移動ロジックを担う。

    - マップデータ（config.FLOOR_MAPS）を保持する。
    - 現在座標 (x, y) から移動可能方向を計算する。
    - 移動後の周辺テキスト描写を生成する。

    Attributes:
        floor        : 現在のフロア番号
        x            : 現在のX座標
        y            : 現在のY座標
        grid         : フロアのグリッドデータ [y][x]
        start        : スタート座標 (x, y)
        goal         : ゴール座標 (x, y)
        fixed_events : 座標 → イベント種別の固定マップ
    """

    def __init__(self, floor: int, x: int, y: int) -> None: ...
    def can_move(self, direction: str) -> bool: ...
    def available_directions(self) -> list[str]: ...
    def move(self, direction: str) -> tuple[int, int]: ...
    def is_goal(self) -> bool: ...
    def is_start(self) -> bool: ...
    def cell_at(self, x: int, y: int) -> int: ...
    def describe_surroundings(self) -> str: ...
    def get_fixed_event(self) -> str | None: ...
```

### 5.2 メソッド仕様

#### `__init__(floor, x, y)`

```python
def __init__(self, floor: int, x: int, y: int) -> None:
    floor_data = FLOOR_MAPS[floor]
    self.floor = floor
    self.x = x
    self.y = y
    self.grid: list[list[int]] = floor_data["grid"]
    self.start: tuple[int, int] = floor_data["start"]
    self.goal:  tuple[int, int] = floor_data["goal"]
    self.fixed_events: dict[tuple[int, int], str] = floor_data.get("fixed_events", {})
```

#### `can_move(direction) -> bool`

```python
def can_move(self, direction: str) -> bool:
    """指定方向へ移動できるか（通路/スタート/ゴール = True、壁/範囲外 = False）"""
    dx, dy = DIRECTIONS[direction]
    nx, ny = self.x + dx, self.y + dy
    if ny < 0 or ny >= len(self.grid) or nx < 0 or nx >= len(self.grid[0]):
        return False
    return self.grid[ny][nx] != CELL_WALL
```

#### `available_directions() -> list[str]`

```python
def available_directions(self) -> list[str]:
    """移動可能な方向名のリストを返す"""
    return [d for d in DIRECTIONS if self.can_move(d)]
```

#### `move(direction) -> tuple[int, int]`

```python
def move(self, direction: str) -> tuple[int, int]:
    """
    指定方向へ移動し、新しい座標 (x, y) を返す。
    移動不可の場合は ValueError を送出する。
    """
    if not self.can_move(direction):
        raise ValueError(f"方向 '{direction}' には移動できません。")
    dx, dy = DIRECTIONS[direction]
    self.x += dx
    self.y += dy
    return self.x, self.y
```

#### `is_goal() -> bool`

```python
def is_goal(self) -> bool:
    """現在地がゴールマス（ボス部屋）かどうか"""
    return (self.x, self.y) == self.goal
```

#### `is_start() -> bool`

```python
def is_start(self) -> bool:
    """現在地がスタートマスかどうか"""
    return (self.x, self.y) == self.start
```

#### `cell_at(x, y) -> int`

```python
def cell_at(self, x: int, y: int) -> int:
    """指定座標のセル種別を返す。範囲外の場合は CELL_WALL を返す"""
    if y < 0 or y >= len(self.grid) or x < 0 or x >= len(self.grid[0]):
        return CELL_WALL
    return self.grid[y][x]
```

#### `describe_surroundings() -> str`

```python
def describe_surroundings(self) -> str:
    """
    現在地の周辺4方向を描写するテキストを返す。

    例:
      北: 壁が立ちはだかっている。
      南: 暗い通路が続いている。
      東: 重厚な扉が見える…何かが待ち構えているようだ。
      西: 壁が立ちはだかっている。
    """
    lines = []
    for direction, (dx, dy) in DIRECTIONS.items():
        cell = self.cell_at(self.x + dx, self.y + dy)
        dir_ja = DIRECTION_NAMES_JA[direction]
        desc   = CELL_DESCRIBE.get(cell, "???")
        lines.append(f"{dir_ja}: {desc}")
    return "\n".join(lines)
```

#### `get_fixed_event() -> str | None`

```python
def get_fixed_event(self) -> str | None:
    """現在地に固定イベントがあれば event_type を返す、なければ None"""
    return self.fixed_events.get((self.x, self.y))
```

## 6. DBスキーマ変更（dungeon_progress）

### 6.1 テーブル変更

`current_x` / `current_y` カラムを追加する。  
デフォルト値は `-1`（未配置を示すセンチネル値）。  
フロア移動時またはダンジョン開始時に `start` 座標で初期化する。  
`map_type == 'linear'` のダンジョンでは `current_x/y` は常に `-1` のまま使用されない。

```sql
-- data/db_init.sql に追加（マイグレーション SQL）
ALTER TABLE dungeon_progress ADD COLUMN current_x INTEGER NOT NULL DEFAULT -1;
```

#### `resolve_event_at(x, y, party, ...) -> EventResult`

```python
def resolve_event_at(
    self,
    x: int,
    y: int,
    party: list,
    hp_mult: float = 1.0,
    atk_mult: float = 1.0,
) -> EventResult:
    """
    座標 (x, y) のイベントを解決する。

    - ゴールマス    : 常にボス戦
    - 固定イベント  : fixed_events の event_type を使用
    - それ以外      : EVENT_WEIGHTS によるランダム選択（既存ロジック再利用）
    """
    mm = MapManager(self.current_floor, x, y)
    if mm.is_goal():
        return self._event_boss(hp_mult, atk_mult)

    fixed = mm.get_fixed_event()
    if fixed:
        return self._resolve_by_type(fixed, party, hp_mult, atk_mult)

    # ランダムイベント（既存の resolve_event を room=0 で流用）
    return self.resolve_event(party, room=0, hp_mult=hp_mult, atk_mult=atk_mult)
```

#### `_resolve_by_type(event_type, ...) -> EventResult`

```python
def _resolve_by_type(
    self,
    event_type: str,
    party: list,
    hp_mult: float,
    atk_mult: float,
) -> EventResult:
    """イベント種別を直接指定してイベント解決を行う（内部メソッド）"""
    if event_type == "encounter":
        return self._event_encounter(party, hp_mult, atk_mult)
    elif event_type == "trap":
        return self._event_trap(party)
    elif event_type == "merchant":
        return self._event_merchant()
    elif event_type == "shrine":
        return self._event_shrine(party)
    elif event_type == "rest":
        return self._event_rest(party)
    elif event_type == "chest":
        return self._event_chest()
    else:
        return EventResult(event_type="nothing", messages=["静かだ…何も起きなかった。"])
```

```python
def render_dungeon_select(user_id: int) -> None:
    """ダンジョン選択UIを描画し、選択されたら session_state を設定する"""
    st.title("🏰 ダンジョン探索")
    st.subheader("挑戦するダンジョンを選んでください")

    with SessionLocal() as db:
        all_dungeons = Dungeon.get_all(db)
        # 旅立ちの洞窟（id=1）のクリア状況を取得
        prog1 = DungeonProgress.get_or_create(db, user_id, 1)
        cleared_tutorial = prog1.is_cleared

    for dungeon in all_dungeons:
        with SessionLocal() as db:
            prog = DungeonProgress.get_or_create(db, user_id, dungeon.id)

        # ロック判定: id=1 は常に開放、id>=2 は旅立ちの洞窟クリアが必要
        is_locked = dungeon.id >= 2 and not cleared_tutorial

        with st.container(border=True):
            col_info, col_btn = st.columns([4, 1])
            with col_info:
                lock_mark = "🔒" if is_locked else ("✅" if prog.is_cleared else "")
                st.markdown(f"**{dungeon.name}** {lock_mark}")
                floor_str = f"{prog.current_floor}F / {dungeon.floor}F"
                status_str = "クリア済み" if prog.is_cleared else floor_str
                st.caption(f"全{dungeon.floor}階層　進行状況: {status_str}")
                if is_locked:
                    st.caption("🔒 旅立ちの洞窟をクリアすると解放されます")
            with col_btn:
                st.text("")  # 縦位置調整
                if st.button("挑戦する", key=f"select_dungeon_{dungeon.id}",
                             disabled=is_locked, use_container_width=True):
                    st.session_state["current_dungeon_id"] = dungeon.id
                    st.rerun()
```

```python
from game.map_manager import MapManager
from config import DIRECTION_LABELS

def render_move_buttons(mm: MapManager) -> str | None:
    """
    十字ボタンを描画し、クリックされた方向名を返す。
    何もクリックされなければ None を返す。
    """
    clicked = None

    _, col_n, _ = st.columns([1, 1, 1])
    with col_n:
        if st.button(DIRECTION_LABELS["north"],
                     disabled=not mm.can_move("north"), key="btn_north"):
            clicked = "north"

    col_w, col_c, col_e = st.columns([1, 1, 1])
    with col_w:
        if st.button(DIRECTION_LABELS["west"],
                     disabled=not mm.can_move("west"), key="btn_west"):
            clicked = "west"
    with col_c:
        st.markdown("<div style='text-align:center'>🧍</div>", unsafe_allow_html=True)
    with col_e:
        if st.button(DIRECTION_LABELS["east"],
                     disabled=not mm.can_move("east"), key="btn_east"):
            clicked = "east"

    _, col_s, _ = st.columns([1, 1, 1])
    with col_s:
        if st.button(DIRECTION_LABELS["south"],
                     disabled=not mm.can_move("south"), key="btn_south"):
            clicked = "south"

    return clicked
```

### 9.5 移動処理フロー

```
ユーザーが方向ボタンをクリック
  │
  ├─ MapManager.move(direction) → 新座標 (nx, ny)
  ├─ session_state["map_x/y"] を更新

  │    └─ event_type="chest"
  │         → ゴールド/アイテム付与 → ログ追加
  │
  ├─ last_event_pos = (nx, ny) を更新
  └─ st.rerun()
```
