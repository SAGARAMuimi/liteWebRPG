## 1. 前提・スコープ

### 現在のレベルアップ実装

```python
# models/character.py
def level_up(self) -> None:
    self.level += 1
    for stat, (mn, mx) in LEVEL_UP_GROWTH.items():
        growth = random.randint(mn, mx)  # ← 全ステータスをランダム成長（固定）
        setattr(self, stat, getattr(self, stat) + growth)
    self.hp = self.max_hp   # レベルアップ時に全回復
    self.mp = self.max_mp

# config.py
LEVEL_UP_GROWTH: dict[str, tuple[int, int]] = {
    "max_hp":  (10, 15),
    "max_mp":  (3,  8),
    "attack":  (1,  3),
    "defense": (1,  2),
}
```

### 課題

- 全キャラクターが同じ確率範囲で成長するため、クラスごとの個性がレベルアップに反映されない
- プレイヤーが関与できないため、成長に対する愛着・戦略性が薄い

### スコープ

- **レベルアップ時に3択の成長プランを提示**し、プレイヤーが選択する
- 選択後に即時反映 → DB 保存 → 画面を更新する
- 既存の `gain_exp()` / `level_up()` のシグネチャ変更は最小限に留め、後退互換を維持する

## 2. 設計方針

### 主な変更点

| 変更箇所 | 変更前 | 変更後 |
|---|---|---|
| `level_up()` の呼び出し | `gain_exp()` 内で自動実行 | レベルアップ検知だけ行い、選択UIに制御を渡す |
| 成長量 | 全ステータス一律ランダム | 選択したプランの係数に従って成長 |
| 戦闘画面 | レベルアップをメッセージで表示して即帰還 | 選択UIを表示してからダンジョンへ戻る |

### 設計の核心：「保留レベルアップ」方式

```
gain_exp() の変更:
  - 経験値が閾値を超えたらレベルを上げる前に「pending_levelups」を設定
  - level_up() はプレイヤーの選択を受け取って呼ぶ

3_battle.py の変更:
  - 勝利後に pending_levelups が残っていれば選択UIを表示
  - 全キャラ分の選択が終わったらダンジョンへ戻るボタンを表示
```

### 4.1 config.py の変更

### 4.2 models/character.py の変更

#### `gain_exp()` の変更

レベルアップ検知だけを行い、`level_up()` の呼び出しを外部に移す。

```python
def gain_exp(self, db: Session, amount: int) -> int:
    """
    経験値を加算し、レベルアップした回数を返す。
    （変更前: bool を返していた → 変更後: int でレベルアップ回数を返す）
    実際の level_up() はプレイヤーの選択後に呼ぶこと。
    """
    self.exp += amount
    count = 0
    while self.exp >= self.level * EXP_PER_LEVEL:
        self.exp -= self.level * EXP_PER_LEVEL
        self.level += 1         # レベル数値だけ先に上げる
        count += 1
    merged = db.merge(self)
    db.commit()
    for attr in ("exp", "level"):
        setattr(self, attr, getattr(merged, attr))
    return count  # 0 = レベルアップなし
```

> **後退互換注意**: `gain_exp()` の戻り値を `bool` から `int` に変更する。  
> `if up:` → `if up > 0:` に変更が必要。テストも合わせて修正する。

#### `level_up()` の変更

プランキーを受け取り、プランの成長量に従ってステータスを上げる。

```python
def level_up(self, plan_key: str = "power") -> dict[str, int]:
    """
    指定プランに従ってステータスを成長させる。
    Returns: 各ステータスの実際の成長量 {"max_hp": 12, "attack": 4, ...}
    プランが不明な場合は "power" にフォールバック。
    """
    from config import LEVEL_UP_PLANS, CLASS_DEFAULT_LEVELUP_PLAN
    if plan_key not in LEVEL_UP_PLANS:
        plan_key = CLASS_DEFAULT_LEVELUP_PLAN.get(self.class_type, "power")
    plan = LEVEL_UP_PLANS[plan_key]
    gains: dict[str, int] = {}
    for stat, (mn, mx) in plan["growth"].items():
        growth = random.randint(mn, mx)
        setattr(self, stat, getattr(self, stat) + growth)
        gains[stat] = growth
    # HP / MP は全回復
    self.hp = self.max_hp
    self.mp = self.max_mp
    return gains
```

#### 勝利時の変更（`battle_result != "win"` ブロック）

```python
if st.session_state.get("battle_result") != "win":
    total_exp  = engine.get_total_exp()
    total_gold = int(sum(e.gold_reward for e in enemies) * diff_cfg["gold_mult"])
    pending: list[dict] = []
    with SessionLocal() as db:
        for chara in party:
            if chara.is_alive():
                count = chara.gain_exp(db, total_exp)  # int を返す（変更後）
                if count > 0:
                    pending.append({
                        "chara_id":   chara.id,
                        "chara_name": chara.name,
                        "class_type": chara.class_type,
                        "count":      count,
                    })
        User.add_gold(db, user_id, total_gold)
    st.session_state["battle_result"]    = "win"
    st.session_state["battle_exp"]       = total_exp
    st.session_state["battle_gold"]      = total_gold
    st.session_state["pending_levelups"] = pending
```

### 複数キャラのレベルアップ

複数キャラが同時にレベルアップした場合、**1キャラずつ順番に選択UI**を表示する。  
（`pending_levelups` リストの先頭から pop していく方式）

```
勇者 → [選択] → ヒロ → [選択] → 全員完了 → ダンジョンへ戻るボタン
```

### 2回以上レベルアップした場合

`count > 1` のとき「2レベルアップ！」と表示し、**同じプランを count 回適用**する。  
（例外: 毎回異なるプランを選ぶ仕様にすると複雑になりすぎるため、1選択で複数回分まとめて適用）
